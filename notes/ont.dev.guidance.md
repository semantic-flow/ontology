## Naming

- Never reuse a class name as a property name with only capitalization changed.
- Prefer explicit relational property names such as `hasDefaultShot` over ambiguous pairs such as `DefaultShot` and `defaultShot`.
- When a property relates one resource to another resource or value, prefer making that relation obvious in the property name rather than relying on case alone.
